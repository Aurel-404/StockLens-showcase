## What It Does

StockLens lets you analyze any publicly traded stock by selecting which analytical sections you want. The app routes each request to the right AI model, fetches live market data from Yahoo Finance, and returns a structured report.

### Analysis Sections

| Section | Model | Web search |
|---|---|---|
| Overview | Claude Sonnet | No |
| Metrics | Yahoo Finance only | No |
| SWOT | Claude Sonnet | No |
| Risks | Claude Sonnet | Up to 2 searches |
| Scenarios | Claude Sonnet | Up to 1 search |
| Guidance Accuracy | Claude Sonnet | Up to 2 searches |
| Fair Value | Claude Sonnet | No |
| Revenue Breakdown | Claude Sonnet | Up to 2 searches |
| Analyst Targets | Yahoo Finance only | No |
| Earnings Calendar | Yahoo Finance only | No |
| Insider Trading | Yahoo Finance only | No |

Sections with model tier "none" skip the Anthropic API entirely and are populated directly from Yahoo Finance data.

### Section Groups and Search Caps

Sections are fetched in 4 parallel group calls. Each group has a hard cap on total web searches:

| Group | Sections | Max web searches |
|---|---|---|
| A | Overview, Metrics | 0 |
| B | SWOT, Risks, Guidance Accuracy | 4 |
| C | Analyst Targets, Scenarios, Insider Trading, Earnings Calendar | 1 |
| D | Fair Value | 0 |

## Key Technical Features

### Section-Based AI Routing

Each section is mapped to a model tier in a central config file:

- **`none`** — Yahoo Finance data injected server-side, no Anthropic call made
- **`haiku`** — lightweight summaries (Changes Since, Portfolio Intelligence)
- **`sonnet`** — deep analytical sections (SWOT, Risks, Fair Value, Scenarios, Guidance Accuracy)

### Multi-Turn AI Loop

The analyze route runs a multi-turn loop: the AI can call the `web_search` tool multiple times within one request. The `max_uses` hard cap is passed directly to the Anthropic API, so the limit is enforced at the platform level.

### Ground-Truth Market Data

The current stock price and shares outstanding from Yahoo Finance are injected into every AI prompt as labeled ground truth. This ensures the AI uses live post-split data rather than potentially stale training-data figures.

### Sector Comparison

- Enter any ticker or sector name to trigger a sector comparison
- For tickers, the app first classifies the primary sector using Yahoo Finance GICS data
- For ambiguous industry tags (e.g. online travel platforms tagged as "Internet Content & Information" by Yahoo), a one-shot Haiku call reads the business description and picks the correct canonical sector
- Sector classifications are cached in Supabase for 30 days with a versioned key
- Every AI-suggested peer ticker is verified against Yahoo Finance — private companies and ETFs are filtered out
- A Supabase-backed generation lock (5-minute TTL) prevents duplicate concurrent generations

### Shared Cache

All fresh analyses are saved to a Supabase-backed shared cache and served to all users. The cache is keyed by ticker + section group, so different users requesting the same analysis get instant results.

### Prompt Caching

The AI system prompt is cached at the Anthropic layer with a 1-hour TTL, reducing input token costs on repeated calls.

### Deterministic Scorecard

Growth, Profitability, Balance Sheet, Valuation, and Momentum scores are computed server-side from raw Yahoo Finance numbers — not generated by the AI.

### Admin Cost Dashboard

Every request is logged to a Supabase table with full metadata: ticker, model used, tokens consumed, actual web searches performed, and estimated cost. An admin-only tab shows daily cost breakdown, model tier usage, section frequency, and recent requests. The API endpoint returns 403 for any non-admin email.

## Development Approach

This project was built using **Claude Code** (Anthropic's AI coding assistant) as a primary development tool alongside standard development practices.

**What AI helped with:**
- Architectural decisions — section routing, group-based search caps, generation lock design, deterministic scorecard logic
- Boilerplate-heavy work — Stripe webhook handling, NextAuth configuration, Supabase schema and query patterns
- Debugging edge cases — post-split price data issues, Yahoo Finance GICS industry mis-tagging for certain tickers
- Iterating on AI prompts for each analysis section to get structured, reliable JSON output

**What I (the developer) owned:**
- All product decisions — what sections to build, what the UX should feel like, what makes a good analysis
- All architectural trade-offs and final design choices
- Feature scope, prioritization, and the overall product vision
- Final review and approval of every implementation
